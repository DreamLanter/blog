从头搭建个人博客

2022-10-16- 2023-08-01
1.4k- 5m
折腾-博客Hexo

前言

从今年年初开始就想搭建一个个人博客，最开始是想用 WordPress 这种偏成熟的解决方案。因为网上资源多

WordPress

用 WordPress 的好处有

  1. 用的人多网上能搜到一堆教程
  2. 有大量免费主题和插件
  3. 方便部署和维护能够随时更新

但在产找过相关资料后权衡利弊最后放弃了，原因 : WordPress 还要选购服务器、域名备案。

这笔费用在日后会成为很大的开销。

WordPress 是动态网站，有独立数据库只用来写博客有点大材小用。

其实就是没钱买服务器

Hexo + GitHub Pages

最后 Blog 选择了 Hexo + GitHub Pages 这套解决方案

为什么要用 Hexo

Hexo 是一个快速、简洁而强大的博客框架，基于 Node.js，同样托管于 GitHub 之上。生态中拥有众多插件主题。你可以基于它快速生成一些静态页面。
你可以使用别人的各种主题与插件，也可以自己定制开发想要的功能。

优点：

  1. 因为主要是用作个人博客，Hexo不需要去购买额外的服务器，可以直接托管到 Github，免费的空间完全够用
  2. Hexo 属于静态网页，体积小，网站性能好
  3. Hexo 关于文章的保存，完全是 md 文档格式，可以更方便的备份和转移，像我这种危机意识比较重的人，特别不放心那些空间商的数据库和空间的安全性，生怕那一天网站崩了就全没了

缺点：

  1. Hexo 基于 Node.js 需要自行配置环境
  2. 不支持原生评论功能

GitHub Pages 部署优点

最大优点微软有💰可以白嫖

GitHub Pages 是最好的——自由、免费、流量够用，目前的访问速度也还不错，完美！

如果会用 Jekyll 可以写好 Markdown 后直接上传，快捷又方便。除了我不会用以外没有任何问题
